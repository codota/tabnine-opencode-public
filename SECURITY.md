# Security Policy

## Reporting a vulnerability

Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.

Instead, report them privately by email to **security@tabnine.com**. Include:

- A description of the vulnerability and its impact.
- Steps to reproduce, or a proof of concept.
- The version affected (`tabnine-opencode --version`) and your platform.

You will receive an acknowledgement of your report. Please give us a reasonable
period to investigate and address the issue before any public disclosure.

## Supported versions

The Tabnine plugin for opencode is in beta. Security fixes are provided for the
most recent release. Please make sure you are on the latest version before
reporting an issue.
